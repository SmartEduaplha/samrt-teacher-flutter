# خطة تحويل مشروع المساعد الشخصي للمعلم إلى Flutter

هذه هي الخطة الشاملة لتحويل منصة الويب الحالية لتطبيق هاتف أصلي (Android/iOS) باستخدام لغة Dart وإطار عمل Flutter. تعتمد الخطة على **5 مراحل أساسية** للتحويل التدريجي:

## المرحلة الأولى: تهيئة المشروع وبنية البيانات
- إعداد بنية المجلدات (Architecture) والتنظيم (Screens, Widgets, Models, Services).
- اختيار وتثبيت أدوات إدارة الحالة (مثل `Riverpod` أو `Provider`).
- إنشاء نماذج البيانات (Models) بلغة Dart.
- إعداد قاعدة البيانات المحلية (Local Storage) باستخدام `Hive` أو `sqflite` ومحاكاة ميكانيكية الـ `localDb` التي كانت في الموقع.

## المرحلة الثانية: المصادقة والتصفح الأساسي (Navigation & Core UI)
- بناء شاشات تسجيل الدخول للمعلم والطالب.
- تصميم الـ Sidebar / Bottom Navigation Bar الأساسي.
- بناء لوحة التحكم الرئيسية (Dashboard).

## المرحلة الثالثة: إدارة الكيانات الأساسية (بوابة المعلم)
- شاشات إدارة المجموعات والطلاب.
- شاشات تسجيل الحضور، المدفوعات، والدرجات.

## المرحلة الرابعة: الميزات المتقدمة (التقارير والتواصل)
- بناء الرسوم البيانية للتقارير (باستخدام مكتبة `fl_chart`).
- واجهة متجر المذكرات وامتحانات الـ Quiz Builder.
- ربط الرسائل الذكية لإرسالها عبر واتساب (`url_launcher`).

## المرحلة الخامسة: بوابة الطالب والإشعارات
- واجهة الطالب لعرض (الدرجات، الحضور، الإشعارات، المتجر الخاص به).
