# ⚖️ Project Governance & Working Protocol

## 🚨 The Golden Rule (Strict Law)
**Strict prohibition from modifying any existing code or adding any new features/files without explicit prior approval from the User.**

## 🛠️ Operational Workflow
1. **Request Analysis**: Every request is first analyzed and researched.
2. **Implementation Plan**: A detailed `implementation_plan.md` must be created for every change, no matter how small.
3. **User Approval**: The developer (AI) must **STOP** and wait for the User to review and approve the plan.
4. **Execution**: Only after receiving a clear "Approved" or "Go ahead" can the developer proceed to apply changes.
5. **Verification**: After implementation, a walkthrough and verification must be provided.

## 📝 Documenting Decisions
- All major architectural decisions must be documented.
- `IMPLEMENTATION_STATUS.md` must be updated as features are completed.
